## 1. 系统概述

lightY 是基于 STM32H750VBT6 (Cortex-M7 @ 480MHz) 的智慧灯杆控制系统，集成环境感知、智能照明、无人机对接、电机控制和 JSON 通信协议。系统无 RTOS，采用协作式多任务调度，通过 USART2 与上位机 (LubanCat) 进行 JSON 双向通信。

### 关键特性

- 无 RTOS 的协作式多任务系统，主循环 + 标志位驱动
- 零 malloc 热路径：调度器使用静态数组，传感器重试表固定 8 槽位
- 中断安全：JSON 解析不在中断上下文中执行，ISR 仅置标志
- 整数优先：调光等关键路径避免浮点运算
- 传感器容错：EMA 滤波 + 失败计数 + 自动重试机制

## 5. 设计目标与约束

1. **无 RTOS**：协作式多任务，通过调度器轮询实现并发
2. **零 malloc 热路径**：调度器使用静态数组，传感器重试表固定槽位
3. **中断安全**：ISR 仅做最小工作（读取数据入队、置标志），JSON 解析在主循环执行
4. **整数优先**：调光关键路径避免浮点，`LightLuxToBrightness()` 使用整数线性公式
5. **PascalCase 命名**：所有公开函数、类型采用 PascalCase
6. **向后兼容**：USART2 JSON 输出格式与 light_show 完全一致，可直接替换

## 6. 主循环与调度器

### 6.1 主循环

```c
void Main(void)
{
    while (1)
    {
        SysTask();           // SOS 按钮状态机 (30ms 消抖)
        SchedulerPoll();     // 协作式调度器

        if (g_ReportFlag)    // TIM15 定时上报
        {
            g_ReportFlag = 0;
            ReportSensor();
        }

        if (g_UnReportFlag)  // USART2/4 命令接收
        {
            g_UnReportFlag = 0;
            UnReport();
        }
    }
}
```

### 6.2 调度器

静态数组协作式调度器，最多 32 个任务，O(1) 查找，无 malloc。

**数据结构：**

```c
typedef int SchedId_t;

typedef enum {
    SCHED_DISABLE = 0,
    SCHED_ENABLE  = 1
} SchedState_t;

typedef struct {
    uint32_t Period;          // 执行周期 (ms)
    uint32_t Deadline;        // 下次执行时间 (ms)
    void (*Entry)(void *arg); // 回调函数
    void *Arg;                // 回调参数
    uint8_t Active;           // 任务是否激活
    uint8_t Run;              // 是否使能执行
} SchedTask_t;
```

**预注册任务：**

| 任务 | 周期 | 功能 |
|------|------|------|
| WeatherTask | 900ms | 读取气象站数据，触发 SendDataToLubanCat |
| WindTask | 1200ms | 读取风速仪数据，触发 SendDataToLubanCat |
| DistanceTask | 100ms | 轮询 4 路距离传感器 |
| LightAutoTask | 500ms | 自动调光算法 |
| SensorRetryTask | 3000ms | 离线传感器重连 |


### 7.5 下行命令

通过 USART2 接收 JSON 命令，格式：

```json
{ "Target": "命令名", "Type": "参数类型", "Data": 值, "SpeakId": 回调ID }
```

- **Target**: 字符串，指定命令名称
- **Type**: 字符串，`"string"` / `"float"` / `"int"` / `"uint"`
- **Data**: 参数值，类型与 Type 一致
- **SpeakId**: 数字或字符串，用于回调标识

执行完成后返回响应（USART2）：

```json
{ "Command":{"SpeakId": 123,"Execute_Status":0}}
```

`Execute_Status`: 0 = 成功，非 0 = 失败。

## 8. 传感器层

### 8.1 通用状态管理

每个传感器维护 `SensorStatus_t`：

```c
typedef enum {
    SensorError  = 0,
    SensorNormal = 1
} SensorState_t;

typedef struct {
    uint8_t      FailedCount;  // 连续失败计数
    SensorState_t State;       // Normal / Error
} SensorStatus_t;
```

**容错机制：**
- 失败阈值 = 4 次连续失败 → 标记 `SensorError`
- 标记后自动注册到重试表（最多 8 个槽位）
- 重试表每 3 秒轮询一次，成功则恢复 `SensorNormal`
- 重试注册 API: `SensorCheckAndRegisterRetry(status, addr, baud, reg)`

### 8.2 EMA 滤波器

```c
uint16_t SensorFilterEma(uint16_t newValue, uint16_t *filteredValue, uint8_t shift);
```

指数移动平均滤波，`shift` 参数控制平滑度（越大越平滑，响应越慢）。

### 8.3 气象站 (Weather)

- Modbus 地址: 0x01, 波特率: 4800
- 寄存器: Humi(500), Temp(501), PM25(503), PM10(504), LuxH(506), LuxL(507)
- 使用 FC03 读取保持寄存器

### 8.4 风速仪 (Wind)

- Modbus 地址: 0x02, 波特率: 9600
- 寄存器: Speed(0x0000), Direction(0x0001)
- 使用 FC03 读取保持寄存器

### 8.5 距离传感器 (Distance)

- Modbus 地址: 0x04-0x07, 波特率: 38400
- 4 通道: Front(0x04), Rear(0x05), Left(0x06), Right(0x07)
- 寄存器: RealValue(0x0001), FilterValue(0x0002), DevAddr(0x0003)
- 每个通道独立维护 `DisDev_t`，包含 Modbus 地址、数据、波特率、状态

### 8.6 INA219 电源监测

- I2C 地址: 灯板 0x40, 车载 0x45
- 测量: 电压 (V)、电流 (A)、功率 (W)
- 校准值: 0x346E

### 8.7 BH1750 光照传感器

- I2C 地址: 0x46
- 提供环境光照度 (lux)

### 8.8 dToF 传感器 (框架)

- 条件编译 `#ifdef DToF_ENABLE`，默认禁用
- 串口可配置 `#define DToF_UART huartX`
- 二进制协议 + CRC16 校验

## 9. 灯光控制与自动调光

### 9.1 基本控制

- PWM 输出: TIM4 CH1 (PD12)
- 亮度范围: 0-1000 (整数)
- 模式: 自动 (`LIGHT_MODE_AUTO = 0`) / 手动 (`LIGHT_MODE_MANUAL = 1`)
- 继电器: PE1 (灯板), PE0 (无线充电)

### 9.2 自动调光算法

```
                    ┌──────────────┐
                    │ 读取光照/距离 │
                    └──────┬───────┘
                           │
                    ┌──────▼───────┐
            ┌───────┤ 光照阈值判断  ├───────┐
            │       └──────────────┘       │
     光照 ≥ 280 lux                  光照 < 250 lux
            │                               │
     ┌──────▼──────┐                ┌───────▼───────┐
     │  关灯 (亮度=0)│                │ 逐传感器更新   │
     └─────────────┘                │ BasicDistance  │
                                    │ 基线 (20次采样)│
                                    └───────┬───────┘
                                            │
                                    ┌───────▼───────┐
                                    │ 快速移动检测   │
                                    │ 变化率 ≥ 50mm?│
                                    └───┬───────┬───┘
                                       是       否
                                        │       │
                                 ┌──────▼──┐ ┌──▼─────────────┐
                                 │最大亮度  │ │距离 < 基线-100?│
                                 │(1000)   │ └──┬──────────┬──┘
                                 └─────────┘   是          否
                                                │           │
                                    ┌───────────▼──┐ ┌──────▼──────────┐
                                    │pow(ratio,0.5)│ │lux线性公式       │
                                    │曲线调光      │ │y=-3.25*lux+1000 │
                                    └──────────────┘ └─────────────────┘
```

### 9.3 基线收敛机制

- 每个传感器独立维护 `BasicDistance[4]`、`BasicDistanceCnt[4]`、`prevDis[4]`
- 连续 20 次采样距离 < 2000mm 后建立基线
- 物体远离时基线自动上移：当前值 > 基线 + 50 时，基线更新为当前值
- 用于消除环境变化（温度漂移、安装松动）对距离检测的影响

### 9.4 光照阈值

- `LIGHT_CLOSE_LUX = 280`：光照 ≥ 此值关灯
- `LIGHT_OPEN_LUX = 250`：光照 < 此值开灯
- 20 lux 回差防止频繁切换

## 15. 引脚变更记录

### v2.0 — I2C 引脚重映射

| 项目 | v1.x | v2.0 | 原因 |
|------|------|------|------|
| I2C SDA | PD10 | PB0 | 解除与升降平台电机冲突 |
| I2C SCL | PD11 | PC5 | 解除与升降平台电机冲突 |
| INA219 | 禁用 | 启用 (0x40/0x45) | 引脚冲突解除 |
| 升降平台 IN1 | — | PD11 | 恢复为电机控制 |
| 升降平台 IN2 | — | PD10 | 恢复为电机控制 |

PD10/PD11 从 I2C 释放后，重新分配给升降平台电机控制。PA5/PA6/PA7 保持 RGB LED 不变。PB0/PC5 配置为开漏输出模式，由软件 I2C 驱动。
